IP地址 115.151.29.58

考察知识重点

地址类别：

A: 1 ~ 127, B: 128 ~ 191, C: 192 ~ 223

(1) 子网掩码的功能

根据子网掩码区分 IP 地址中的网络号和主机号，

根据子网掩码划分 IP 地址，全 1 代表网络位，全 0 代表主机号。

(2) 受限广播地址

地址形式唯一，即 32 位全为 1 的 IP 地址

或 (255.255.255.255)

(3) 直接广播地址 网络位不变，主机位变为 1

(4) “主机号”地址 网络位全变成 0，主机位不变

(5) 网络地址： 网络位不变，主机号为 0

(6) 回送地址：

A 类 I P 地址中的 127.0.0.0 是一个保留地址，它即回送地址。

用于网络软件测试和本地进程间通信使用。

(7) 子网内第一个可用的 IP 地址：网络地址（十进制最后一位数）+1

(8) 子网内最后一个可用的 IP 地址：直接广播地址（十进制最后一位数）-1

综合题第二道：特殊题型

题型一：考察交换机catalyst 6500系列的VLAN（虚拟网络）的配置

链接：计算机三级 网络技术 大题 第二题 类型2_1. 如图所示,有2台计算机连接在一台catalyst 6500交换机上,它们分别被划分在id号-CSDN博客

这类题目考查的是交换机Catalyst 6500系列VLAN的配置。

交换机VLAN配置的主要任务是：

配置VTP，

建立和删除VLAN，

为交换机端口分配VLAN

在交换机端口上配置VLAN Trunk。

  • （1）交换机VTP的配置 配置VTP的任务主要有两个，一是建立VTP域，二是设置VTP的工作模式。

    VTP有三种工作模式：VTP Server、VTP Client、VTP Transparent。

    其中

    VTP Server维护该VTP域中所有VLAN信息列表，可以建立、删除或修改VLAN；

    VTP Client也维护VTP域中所有VLAN信息列表，但其VLAN的配置信息是从Server模式学到的，且不能建立、删除、修改VLAN，

    VTP Transparent不参与VTP工作，不从Server模式学习VLAN的配置信息，而拥有本设备上自己维护的VLAN信息，可以建立、删除和修改本机上的VLAN信息。

    区别：VTP Server和VTP Transparent都能够建立、删除和修改VLAN信息，但是VTP Client不能，只能从Server中学习。

    进入配置界面的命令

    ①利用enable命令进入配置界面，配置VTP的域名和工作模式；

    ②配置VTP域名：Catalyst 6500>(enable) set VTP domain<域名>；

    ③配置VTP工作模式：Catalyst 6500>(enable) set VTP mode<工作模式名称>。

  • （2）建立或删除VLAN

    ①建立VLAN语句是：Catalyst 6500>(enable) set VLAN name< VLAN_ name >；

    ②删除VLAN语句是：Catalyst 6500>(enable) clear VLAN。

  • （3）为交换机端口分配VLAN 为端口分配VLAN:

    Catalyst 6500>(enable) set VLAN<端口号>。

  • （4）VLAN trunk的配置

    trunk是中继模式

  • VLAN trunk的配置任务有：设置交换机端口的VLAN Trunk模式，

  • 为VLAN Trunk封装VLAN协议和设置VLAN Trunk运行中继的VLANs。

    ①配置VLAN Trunk模式，封装VLAN协议。

    Catalyst 6500>(enable) set trunk<端口号>on<封装协议类型>；

    （封装协议IEEE802.1Q要写成dot1q）

    ②设置允许中继的VLAN Catalyst 6500>(enable) set trunk<端口号>VLAN< VLAN ID >

考点 3，英语翻译（题目中直接有答案，只需要自己找到对应上就行）

  • Client address (客户端地址)：这是指分配给客户端设备的IP地址。在DHCP交互过程中，客户端通过DHCP请求获得这个地址，并将其设置为自己的网络接口地址。

  • Subnet mask (子网掩码)：这是一个与IP地址配合使用的32位数字，用于确定IP地址中的网络部分和主机部分。它帮助设备识别哪些IP地址属于同一个子网，从而进行正确的数据包路由。

    例如，一个常见的C类地址的子网掩码是255.255.255.0，表示前24位是网络地址，后8位是主机地址。

  • Gateway address (网关地址) = 默认网关

    Default Gateway: 10.1.1.1：这是指网络中的默认网关设备的IP地址，通常是一个路由器。

    当设备需要发送数据到其本地子网以外的目的地时，它会将数据包发送到默认网关，由网关负责转发到正确的网络。

    在这个例子中，默认网关的地址是10.1.1.1。

这些参数都是DHCP服务器在响应客户端的请求时提供的重要网络配置信息，确保客户端能够正确地在网络中通信。

如果将172.0.35.128/25划分成3个子网，其中第一个子网能容纳55台主机，另外两个子网分别能容纳25台主机。

解题步骤：

  1. 先观察子网掩码 /25，求出主机位。

    32-25=7（主机位）

    通过主机位求出可分配的主机数（2的n次方减2）2^7=128 128-2=126

  2. 计算 2^n-2 ≥ 容纳主机数 求出几（主机位，掩码末尾0的个数）

    总：126

    55台 2^6-2=64-2=62, n=6 最后一位1100 0000 /26

    25台 2^5-2=32-2=30, n=5 最后一位1110 0000/27

    IP地址段的起始地址和结束地址间必须用减号“-”，中间不能有空格
